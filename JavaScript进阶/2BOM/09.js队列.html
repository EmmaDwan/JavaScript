1.JavaScript是单线程的 ———— 同一时间只能做一件事。单线程意味着，所有任务都要排队。 如果JS执行时间过长，会造成页面渲染不连贯，导致加载阻塞感。 为了解决此问题，利用多核CPU的计算能力，HTML5提出Web Worker标准，循序JavaScript脚本创造多个线程 所以JS中出现了同步和异步 2. (1)同步: 前一个任务结束，执行下一个 程序执行顺序 = 人物排列顺序 (2)异步: 某件事耗时长，做这件事的同时，处理其他事情 3.JS把各任务分为 同步任务和异步任务 (1)同步任务放入主线执行栈 (2)异步任务是回调函数，放入任务队列(消息队列)，比如 普通事件: click resize 资源加载: load error 定时器: setInterval,setTimeout 4.JS执行顺序 (1)执行 主线执行栈的 (2)遇到回调函数，放入任务队列 (3)所有同步任务执行完毕后，依次读取任务队列中的异步任务，这些异步任务进入执行栈 click事件，触发后 才会放进任务队列， 计时器，时间到了 才放进任务队列 5.事件循环event loop JS会反复在 主线执行栈 和 任务队列，来回查看有没有新任务要执行， 即主线执行栈不断重复获得任务、执行任务，再获取任务、再执行。
